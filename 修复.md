## 1. 硬编码凭据（Hardcoded Credentials） 🔴

### 发现过程

1. **代码审计**：打开 `app.py`，直接翻阅源代码，在第 6-23 行发现用户凭据以字典形式直接写在代码中。
2. **Git 历史检查**：若项目使用 Git，执行 `git log -p` 可追溯凭据何时被提交，且无法从历史中彻底删除。
3. **源码泄露场景**：通过任意文件读取漏洞或 `.git` 目录暴露，攻击者可获取完整源码，从而拿到所有账号密码。

### 漏洞位置

`app.py` 第 6-23 行

```python
USERS = {
    "admin": {
        "username": "admin",
        "password": "admin123",
        ...
    },
    "alice": {
        "username": "alice",
        "password": "alice2025",
        ...
    }
}
```

### 漏洞描述

用户名和密码直接以硬编码形式写在源代码中。任何能接触到源代码的人都能直接得知所有用户的凭据。凭据随代码提交到版本控制系统后，将永久保留在 Git 历史中。

### 危害

- 源码泄露即等于所有账号泄露
- 无法单独更新凭据而不修改代码
- 凭据出现在版本控制历史中，难以彻底清除

### 修复方案

将用户数据迁移到数据库，密码使用哈希存储：

```python
import sqlite3
import hashlib
import os

def get_db():
    db = sqlite3.connect('users.db')
    db.row_factory = sqlite3.Row
    return db

def hash_password(password: str) -> str:
    salt = os.urandom(32)
    key = hashlib.pbkdf2_hmac(
        'sha256', password.encode('utf-8'), salt, 100000
    )
    return salt.hex() + ':' + key.hex()

def verify_password(password: str, stored: str) -> bool:
    salt_hex, key_hex = stored.split(':')
    salt = bytes.fromhex(salt_hex)
    key = bytes.fromhex(key_hex)
    new_key = hashlib.pbkdf2_hmac(
        'sha256', password.encode('utf-8'), salt, 100000
    )
    return new_key == key
```

## 2. 明文存储密码 🔴

### 发现过程

1. **查看认证逻辑**：审计 `app.py` 中的登录处理代码（第 41 行），发现密码比较是直接的字符串等值判断 `USERS[username]["password"] == password`。
2. **数据存储检查**：查看 `USERS` 字典，密码字段 `"admin123"`、`"alice2025"` 均是原始明文，未经任何哈希处理。
3. **验证方式**：尝试用正确的用户名 "admin" 和密码 "admin123" 登录系统，确认认证通过——说明系统直接比对明文密码。

### 漏洞位置

`app.py` 第 6-23 行（数据定义）与第 41 行（认证逻辑）：

```python
"password": "admin123",   # 明文存储
"password": "alice2025",  # 明文存储
```

```python
if username in USERS and USERS[username]["password"] == password:
```

### 漏洞描述

用户密码以明文形式存储，认证时直接进行字符串比较。这违反了 OWASP Top 10 中"加密失败"（Cryptographic Failures）的安全要求。

### 危害

- 数据库/源码泄露时攻击者直接获得所有密码
- 用户可能在不同平台使用相同密码，导致"撞库"攻击
- 内部人员可随时查看他人密码

### 修复方案

使用 bcrypt / argon2 / PBKDF2 等单向哈希算法存储密码：

```python
import bcrypt

# 存储密码时
def create_user(username, password):
    hashed = bcrypt.hashpw(password.encode('utf-8'), bcrypt.gensalt())
    # 将 hashed 存入数据库

# 验证密码时
def check_password(username, password):
    stored_hash = get_user_password_hash(username)
    return bcrypt.checkpw(password.encode('utf-8'), stored_hash)
```

## 3. HTML 注释泄露凭据 🟠

### 发现过程

1. **查看页面源代码**：在浏览器中打开登录页面（`/login`），右键选择"查看页面源代码"（或按 `Ctrl+U` / `F12`）。
2. **搜索关键字段**：在源码中搜索 `password`、`admin`、`账号`、`密码` 等关键词。
3. **定位泄露点**：在 `templates/login.html` 第 1 行发现 HTML 注释，直接暴露了管理员账号和密码。

### 漏洞位置

`templates/login.html` 第 1 行

```html
<!-- 调试信息 - 默认管理员账号 用户名: admin 密码: admin123 -->
```

### 漏洞描述

登录页的 HTML 注释中包含了管理员的默认用户名和密码。任何人查看页面源代码即可直接看到凭据。

### 危害

- 任何访问登录页的用户都能获取管理员凭据
- 搜索引擎可能缓存包含凭据的页面
- 是信息泄露的典型案例

### 修复方案

删除 HTML 注释中的敏感信息。如确需调试信息，应在开发环境通过日志输出，且生产环境自动关闭：

```html
<!-- 删除此行注释 -->
```

安全的调试方式：
```python
import logging
logging.basicConfig(level=logging.DEBUG if DEBUG else logging.WARNING)
logging.debug("管理员账号初始化完成")  # 不包含凭据本身
```

## 4. 前端明文展示密码 🟠

### 发现过程

1. **正常登录**：以 `admin` 身份登录系统（用户名: `admin`，密码: `admin123`）。
2. **查看个人信息页**：登录成功后跳转到首页，个人信息区域直接显示了密码明文 `admin123`。
3. **检查模板代码**：审计 `templates/index.html` 第 10 行，确认 `{{ user.password }}` 直接渲染了密码原始值。
4. **验证信息泄露渠道**：
   - 浏览器开发者工具 → 网络面板 → 查看 HTML 响应体，密码明文在网络传输中出现
   - 浏览器历史记录/缓存中保留该页面，密码明文可被恢复

### 漏洞位置

`templates/index.html` 第 10 行

```html
<li><span class="label">密码：</span>{{ user.password }}</li>
```

### 漏洞描述

用户登录后，个人信息页面直接以明文显示密码。密码应该是"只写"的，永远不应在 UI 中回显。

### 危害

- 肩窥攻击（旁人可见屏幕上的密码）
- 浏览器历史/缓存可能保留密码明文
- XSS 攻击可轻易窃取密码

### 修复方案

不在前端展示密码字段，后端也避免将密码传递给模板：

```html
<!-- 直接移除密码行，或显示为掩码 -->
<li><span class="label">密码：</span>********</li>
```

```python
# 在渲染模板前移除密码字段
user_info = {k: v for k, v in USERS[username].items() if k != "password"}
return render_template("index.html", username=username, user=user_info)
```

## 5. 弱密码策略 🟡

### 发现过程

1. **收集用户凭据**：通过代码审计获取硬编码的用户密码列表。
2. **密码强度评估**：
   - `admin` 的密码 `admin123`：仅 8 位，包含常见单词，属于 Top 100 弱密码
   - `alice` 的密码 `alice2025`：包含用户名本身，仅追加了年份数字
3. **尝试弱密码登录**：使用常见弱密码字典（如 `rockyou.txt`）对登录接口进行测试，验证 `admin123` 等密码的可爆破性。
4. **确认无强度限制**：检查注册/修改密码相关代码，确认系统没有任何密码复杂度要求。

### 漏洞描述

- `admin` 的密码 `admin123` 是极常见的弱密码
- `alice` 的密码 `alice2025` 包含用户名，容易被猜测
- 系统没有密码复杂度要求（长度、大小写、特殊字符等）
- 没有密码过期/强制修改机制

### 危害

- 容易被暴力破解和字典攻击
- 弱密码在社工攻击中非常脆弱

### 修复方案

实施密码强度策略：

```python
import re

def validate_password(password: str) -> tuple[bool, str]:
    """
    验证密码强度，返回 (是否合法, 错误信息)
    """
    if len(password) < 8:
        return False, "密码长度至少为 8 位"
    if not re.search(r"[A-Z]", password):
        return False, "密码必须包含至少一个大写字母"
    if not re.search(r"[a-z]", password):
        return False, "密码必须包含至少一个小写字母"
    if not re.search(r"[0-9]", password):
        return False, "密码必须包含至少一个数字"
    if not re.search(r"[!@#$%^&*(),.?\":{}|<>]", password):
        return False, "密码必须包含至少一个特殊字符"
    # 检查是否包含常见弱密码
    common_passwords = {"password", "123456", "admin123", "qwerty"}
    if password.lower() in common_passwords:
        return False, "密码太常见，请更换"
    return True, ""
```

## 6. 无登录频率限制（暴力破解） 🟡

### 发现过程

1. **抓包分析**：使用 Burp Suite 或浏览器开发者工具，截获 `/login` 的 POST 请求，确认请求格式（`username` 和 `password` 参数）。
2. **编写爆破脚本**：使用 Python 或 Burp Intruder，对登录接口连续发送大量请求，每次尝试不同的密码组合。
3. **验证无限次尝试**：连续发送 100+ 次错误密码请求，观察系统是否返回限流提示或锁定账号——结果发现系统未做任何限制，每次均返回正常的"用户名或密码错误"。
4. **字典攻击验证**：使用 `rockyou.txt` 等常见密码字典，自动化遍历，短时间内即可命中弱密码。

### 漏洞描述

`/login` 路由没有限制登录尝试次数。攻击者可以使用自动化工具进行暴力破解，无限尝试用户名和密码组合。

### 危害

- 弱密码账号可被快速爆破
- 消耗服务器资源
- 配合弱密码策略，攻击成功率极高

### 修复方案

使用 Flask-Limiter 实现速率限制：

```python
from flask_limiter import Limiter
from flask_limiter.util import get_remote_address

limiter = Limiter(app=app, key_func=get_remote_address)

@app.route("/login", methods=["POST"])
@limiter.limit("5 per minute")  # 每分钟最多 5 次尝试
def login():
    ...
```

或自行实现登录失败计数器：

```python
from collections import defaultdict
from datetime import datetime

login_attempts = defaultdict(list)  # {ip: [timestamp, ...]}

def is_rate_limited(ip: str) -> bool:
    """检查 IP 是否被限流"""
    now = datetime.now()
    login_attempts[ip] = [
        t for t in login_attempts[ip]
        if (now - t).total_seconds() < 60
    ]
    return len(login_attempts[ip]) >= 5

@app.route("/login", methods=["POST"])
def login():
    ip = request.remote_addr
    if is_rate_limited(ip):
        return render_template("login.html", error="登录尝试过于频繁，请 1 分钟后再试")
    login_attempts[ip].append(datetime.now())
    ...
```

## 总结

本靶场涵盖了 Web 开发中最常见的密码安全漏洞。开发者应当遵循以下原则：

1. **永远不要**在代码或注释中硬编码密码
2. **永远不要**明文存储密码——使用 bcrypt/argon2 哈希存储
3. **永远不要**在页面上回显密码明文——密码应"只写不读"
4. 实施**密码强度策略**，拒绝弱密码
5. 添加**登录频率限制**，防止暴力破解
6. 密码凭据仅通过**安全渠道**（环境变量、密钥管理服务）传递
